1. OpenAI (GPT + Embeddings)

Where: Used across the entire backend. The AI engine behind nearly every smart feature in Keryx.

Why: Provides the intelligence layer. Without OpenAI, Keryx would be a simple note-taking app. It turns free-form voice or text into structured, searchable, actionable data.

When it activates:

  • Every time you save a memory (extracts topics, mood, people, importance, categories)
  • Every search query (generates embeddings for semantic matching)
  • Morning briefings (synthesizes recent data into a personalized summary)
  • Pattern detection (spots recurring themes in your memories)
  • Goal analysis and milestone suggestions
  • Thematic insights and deep pattern analysis with Q&A
  • Intent detection (determines if input is a log or a search query)
  • AI action detection (spots calendar events, emails, reminders in your input)
  • People duplicate detection and AI-powered people search
  • Financial query answering (when Plaid is connected)
  • Personal news feed generation
  • Contextual discoveries (generates search queries from your data)
  • Message conversation analysis (topics, mood, importance)
  • Ideas workspace AI assistance
  • Personal insights generation

How it works:

  • Uses GPT-4o-mini for fast, cost-effective processing of most tasks
  • Uses text-embedding-3-small for generating vector embeddings (1536 dimensions)
  • Connects via Replit AI Integration (preferred) or direct OpenAI API key as fallback
  • Rate-limited: 15 requests per minute per user on AI routes
  • 30-minute TTL cache for briefings and alerts to avoid redundant calls
  • 30-second timeout with 2 automatic retries

Key files:

server/ai-service.ts, server/message-ai-service.ts, server/ai-actions-service.ts, server/contextual-discoveries-service.ts

Active Secrets: OPENAI_API_KEY

2. Google Calendar

Where: Settings page (connection toggle), memory creation flow, morning briefings, AI actions, dashboard.

Why: Automatically links your memories to calendar events. If you log "the team meeting went well," Keryx checks your calendar and attaches the actual meeting details. Also lets AI create calendar events from natural language.

When it activates:

  • When saving a memory: Searches for calendar events happening around the same time and auto-links matching ones
  • During morning briefings: Pulls today's schedule into the briefing summary
  • AI action execution: Creates calendar events when you say things like "schedule a dentist appointment for Thursday at 2pm"
  • Contextual discoveries: Calendar events inform search queries for relevant content
  • Personal insights: Calendar data contributes to pattern analysis
  • Duplicate event checking before creating new ones

How it works:

  • Connected via Replit's Google Calendar connector (OAuth managed by Replit)
  • Uses Google Calendar API v3 via googleapis library
  • Tokens cached for 30 seconds to ensure freshness
  • Searches events within +/- 2 hours of memory timestamp for auto-linking
  • Handles both all-day events and timed events correctly across timezones
  • Connection status checked dynamically; gracefully degrades when disconnected

server/calendar-service.ts

8. Tavily AI Search

Where: Contextual Discoveries page, accessible from the main navigation.

Why: Makes Keryx proactive rather than just reactive. Instead of you searching the web, Keryx analyzes your memories, calendar, emails, and finances to find content that's relevant to YOUR life right now. Planning a trip? It finds travel tips. Mentioned a health concern? It surfaces helpful articles.

When it activates:

  • When you visit the Discoveries page: Generates personalized content based on recent data
  • Analyzes recent memories (last 7 days), upcoming calendar events, email highlights, financial patterns, and location history
  • Results filtered by relevance score (minimum 0.6 threshold)
  • Cached for 30 minutes to avoid excessive API calls

How it works:

  • Step 1: GPT analyzes your recent data to identify "insight contexts" (upcoming trip, recurring topic, notable spending, etc.)
  • Step 2: GPT generates targeted web search queries from those insights
  • Step 3: Tavily executes the searches and returns results with relevance scores
  • Step 4: Results are categorized (travel, shopping, professional, lifestyle, financial, local, general)
  • Urgency levels: immediate, upcoming, or general
  • Notable transaction threshold: $100+ triggers financial discoveries

9. Location Services

Where: Memory creation (automatic capture), Location History page, Google Timeline import, frequent places detection.

Why: Adds "where" context to your memories. Know not just what you were thinking, but where you were. Builds a history of your movements and identifies places you visit frequently (home, office, gym, etc.).

When it activates:

  • When saving a memory: If location permission is granted, captures GPS coordinates and attaches them
  • Google Timeline import: Bulk import location history from Google Takeout
  • Frequent place detection: Clusters nearby locations to identify places you visit regularly
  • Reverse geocoding: Converts GPS coordinates into readable addresses
  • Location-based reminders: Triggers reminders when you're near a specific place

How it works:

  • Frontend: Uses Browser Geolocation API (navigator.geolocation) for GPS capture
  • Reverse geocoding: OpenStreetMap Nominatim API (free, no API key needed)
  • Rate limited: 1 request per second to comply with Nominatim usage policy
  • Location clustering: Groups locations within proximity radius to detect frequent places
  • Stores history in location_history table, frequent places in frequent_places table
  • Sources tracked: "memory" (automatic capture), "import" (Google Timeline), "manual"

18. Database (Neon PostgreSQL)

Why: The persistent backbone of the entire system. Stores memories, people, settings, AI actions, location history, financial data, messages, ideas, goals, reminders, push subscriptions, and AI cache.

When it activates:

  • Every single API request reads from or writes to the database
  • Session management stored in PostgreSQL
  • Vector embeddings stored for semantic search

How it works:

  • Neon serverless PostgreSQL (managed by Replit)
  • Drizzle ORM for type-safe queries and schema management
  • WebSocket connections via @neondatabase/serverless
  • Schema migrations via npm run db:push (never manual SQL)
  • All queries filter by userId for data isolation
  • Strategic indexes on frequently queried columns
  • JSONB columns for flexible metadata storage
  • Vector type columns for embedding storage (1536 dimensions)

Key tables:

  • settings - Per-user configuration

  • aiActions, aiActionPreferences - AI task management
  • ai_cache - Cached AI responses (30-min TTL)
  • location_history, frequent_places - Location data
  • pushSubscriptions - Push notification endpoints
  • ideas, ideaTasks - Workspace items
  • goals, reminders - Tracking systems
  • messageConversations, messages, messageImports - SMS data

19. Authentication

Where: Login/signup pages, every API route (session validation).

Why: Protects your data. Every piece of information in Keryx is tied to your account and invisible to other users. No one can access your memories, people, finances, or messages without being logged in as you.

When it activates:

  • On every API request: Session cookie validated via Passport.js middleware
  • On login/signup: Password hashed with bcrypt, session created
  • Companion app: Session-based authentication for API calls

How it works:

  • Passport.js with local strategy (username/password)
  • Passwords hashed with bcrypt (10 rounds)
  • Sessions stored in PostgreSQL via connect-pg-simple
  • Session cookie sent with every request for stateful authentication
  • All database queries include userId filter to prevent cross-user data access
  • Object ownership validated before any update/delete operation
  • Zod input validation on all request bodies

20. Chrome Extension / Live SMS Relay

Where: A Chrome Extension (MV3) installed in your browser. Runs alongside Google Messages for Web. Relay API key configured in Settings → Integrations.

Why: Unlike the SMS import (which is a one-time batch upload), the Chrome Extension relays new messages to Keryx in real time. As you receive and send texts inside Google Messages for Web, the extension automatically forwards them to Keryx. No manual exports, no delays — your message history stays continuously up to date.

When it activates:

  • While you have Google Messages for Web open in Chrome: Observes new message bubbles as they appear in the DOM
  • Only intercepts messages added after the extension session started (ignores the pre-loaded conversation history to avoid duplicate imports)
  • Fires on both incoming (received) and outgoing (sent) messages
  • Handles rapid multi-message sequences by processing them one at a time via a serialized queue

How deduplication works:

  • Each relayed message is assigned a content-based ID derived from: sender address + direction + hour-bucket + first 80 characters of message body
  • A 24-hour local cache in chrome.storage.local tracks relayed message IDs to prevent the same message from being sent twice within the same session or across page reloads
  • On the server side, the same content-based ID is used as the externalId for database deduplication — so even if the extension sends a message twice, only one record is stored

How it works (technical flow):

  • A MutationObserver in the content script watches for new message nodes added to the Google Messages DOM
  • When a new message bubble is detected, the extension reads the contact address, message body, direction, and timestamp
  • The message is queued and sent via POST /api/relay/inbound using your Relay API key (X-API-Key header)
  • Keryx stores the message and routes it into the Messages page — the same view used for SMS archive imports
  • The extension popup shows a green/red connection indicator so you can confirm it's active and connected

Installation steps

  1. In Keryx Settings → Integrations, copy your Relay API key (starts with rky_)
  2. Download the extension folder from your Keryx project (extension/ directory)
  3. Open Chrome → chrome://extensions → enable Developer mode
  4. Click Load unpacked and select the extension/ folder
  5. Open the extension popup, paste your Relay API key, and click Save
  6. Navigate to messages.google.com/web and open any conversation
  7. The status light should turn green — new messages will relay automatically

Updating the extension

  • When the extension files are updated (e.g., after a bug fix), return to chrome://extensions and click the reload/refresh icon on the Keryx extension card
  • Current version: 1.0.2
  • Your API key is saved in chrome.storage.local and persists across reloads — you only need to enter it once

Deleting relayed messages

  • In the Messages page, open any conversation and tap the trash icon in the top-right to remove all relay-sourced messages for that conversation
  • This is useful if you want to reset a conversation after reconfiguring the extension

Key files:

extension/content.js, extension/background.js, extension/popup.js, server/routes.ts (POST /api/relay/inbound)

Active Chrome only (MV3) Requires: Relay API key from Settings No external service
