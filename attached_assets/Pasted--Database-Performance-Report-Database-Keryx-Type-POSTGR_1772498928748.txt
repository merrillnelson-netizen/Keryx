# Database Performance Report

**Database:** Keryx  
**Type:** POSTGRESQL  
**Analysis Date:** 2026-03-03 00:47 UTC  
**Health Grade:** D  
**Summary:** Serious issues found with many missing FK indexes and high table bloat impacting performance.  
**Data Sources:** pg_stat_user_indexes + pg_stat_user_tables, pg_stat_user_tables  

## Critical Findings

- 26 foreign key columns without supporting indexes, e.g., log_entries.user_id → users(id)
- High table bloat in session with 4100.0% dead rows, never vacuumed
- High sequential scan ratio in users table with 34857 seq_scans and only 3 index_scans
